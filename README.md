 Отчёт о выполнении лабораторной работы № 20 — миграция данных
Цель работы
Реализовать механизм миграции данных для приложения TaskTrackerWORK: нормализовать данные задач при запуске приложения и автоматически сохранить изменения в файле tasks.json.

Что сделано
Создана модель данных TaskItem (Core/Models/TaskItem.cs):

поля: Id, Title, Description, Status;

значения по умолчанию для строк: "".

Реализован сервис нормализации TaskNormalizer (Core/Migration/TaskNormalizer.cs):

замена null в Title и Description на пустую строку "";

удаление пробелов по краям (Trim) для Title и Description;

обрезка длины: до 50 символов для Title, до 200 символов для Description.

Интегрирована миграция в запуск приложения (Program.cs):

проверка существования файла tasks.json (создание пустого массива, если файла нет);

загрузка данных из JSON;

применение нормализации ко всем задачам;

сохранение файла только при наличии изменений;

вывод результата в консоль.

Подготовлен тестовый JSON для проверки всех сценариев нормализации.
